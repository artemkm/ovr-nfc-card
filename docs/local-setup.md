# Локальный запуск проекта

Эта инструкция описывает, как поднять MVP ОВР NFC на локальном компьютере с
нуля. Для локального запуска не нужен Yandex Cloud: тестовые данные хранятся в
JSON-файлах внутри проекта.

## Что нужно установить

- Git.
- Node.js версии 22 или новее.
- npm. Он устанавливается вместе с Node.js.
- Терминал: Terminal на macOS, PowerShell на Windows или любой shell на Linux.

## 1. Проверить Node.js и npm

Откройте терминал и выполните:

```bash
node -v
npm -v
```

Node.js должен быть версии 22 или новее. Подойдут, например:

```text
v22.0.0
v24.0.0
v26.0.0
```

Если Node.js не установлен или версия ниже 22, установите актуальную версию с:

```text
https://nodejs.org/
```

После установки закройте и заново откройте терминал, затем снова проверьте
`node -v`.

## 2. Скачать проект

Выберите папку, где будет лежать проект, и выполните:

```bash
git clone git@github.com:artemkm/ovr-nfc-card.git
cd ovr-nfc-card
```

Если SSH-доступ к GitHub не настроен, используйте HTTPS:

```bash
git clone https://github.com/artemkm/ovr-nfc-card.git
cd ovr-nfc-card
```

## 3. Установить зависимости

Node.js-приложение лежит не в корне, а в папке `functions/ovr-nfc-card`.

Перейдите в нее и установите зависимости:

```bash
cd functions/ovr-nfc-card
npm install
```

После этого локально появится папка `node_modules`. В Git она не коммитится.

## 4. Проверить код

Перед запуском можно выполнить синтаксическую проверку:

```bash
npm run check
```

Если ошибок нет, команда завершится без сообщений об ошибках.

## 5. Запустить локальный сервер

В той же папке `functions/ovr-nfc-card` выполните:

```bash
npm run dev
```

Ожидаемый вывод:

```text
OVR NFC MVP is running at http://localhost:3000/admin
```

Пока вы работаете с локальной версией, это окно терминала должно оставаться
открытым.

## 6. Открыть админку

Откройте в браузере:

```text
http://localhost:3000/admin
```

Локальный логин:

```text
admin
```

Локальный пароль:

```text
admin
```

## 7. Что можно проверить в админке

После входа должен открыться список тестовых участников.

В локальной админке можно:

- искать участников по ФИО, членскому номеру, специальности, должности или городу;
- открыть карточку участника;
- сгенерировать или перегенерировать NFC-ссылку;
- скопировать NFC-ссылку без `http://` или `https://`;
- загрузить, заменить или удалить фотографию участника;
- посмотреть QR-код визитки.

## 8. Открыть публичную визитку

В админке откройте участника, у которого есть NFC, и скопируйте NFC-ссылку.

Для локальной проверки при необходимости добавьте перед ней `http://`. Пример:

```text
http://localhost:3000/c/example-token
```

Публичная визитка должна показывать:

- ФИО;
- логотип Общества врачей России;
- статус членства;
- научное звание и научную степень;
- должность;
- город;
- публичный телефон и email;
- фотографию, если она загружена;
- ссылку "Добавить в контакты";
- QR-код визитки.

## 9. Где хранятся локальные данные

Исходные тестовые данные лежат здесь:

```text
data/sample-members.json
```

При первом локальном запуске приложение создает рабочую локальную базу:

```text
data/local-db.json
```

`data/local-db.json` используется только локально и не коммитится в Git.

Загруженные локальные фотографии лежат здесь:

```text
data/uploads/
```

Эта папка тоже не коммитится в Git.

## 10. Как сбросить локальную базу

Остановите сервер через `Ctrl+C`.

Из корня проекта выполните:

```bash
cp data/sample-members.json data/local-db.json
```

Затем снова запустите сервер:

```bash
cd functions/ovr-nfc-card
npm run dev
```

Это сбросит данные участников. Уже загруженные файлы в `data/uploads/` команда
не удаляет.

## 11. Как остановить сервер

В окне терминала, где запущен `npm run dev`, нажмите:

```text
Ctrl+C
```

## 12. Если порт 3000 занят

Если при запуске появляется ошибка `EADDRINUSE`, значит порт `3000` уже занят
другим процессом.

На macOS или Linux можно найти процесс так:

```bash
lsof -nP -iTCP:3000 -sTCP:LISTEN
```

После этого остановите старый процесс или закройте другое окно терминала, где
уже запущен сервер.

## 13. Частые проблемы

### `npm: command not found`

Node.js/npm не установлен или терминал не был перезапущен после установки.
Установите Node.js и заново откройте терминал.

### `Cannot find module`

Не установлены зависимости. Выполните:

```bash
cd functions/ovr-nfc-card
npm install
```

### Админка открылась, но данные старые

Локальная версия читает `data/local-db.json`. Если нужно вернуть тестовые данные
к исходному состоянию, сбросьте файл из `data/sample-members.json`.

### Фото пропали после переноса проекта

Локальные фото лежат в `data/uploads/` и не хранятся в Git. Если проект был
перенесен на другой компьютер, фотографии нужно загрузить заново через админку.

## 14. Чем локальная версия отличается от Яндекса

Локальный режим:

```text
index.js -> локальный Node.js сервер -> data/local-db.json
```

Yandex Cloud режим:

```text
cloud.js -> Cloud Function -> YDB
```

Роутинг, HTML-шаблоны и основная логика общие для обоих режимов. Отличается
только хранилище данных.
